# 7.5. Updating Documentation

## 7.5.1. Updating the Guide

The MacPorts Guide documentation is maintained in a [Git repository on GitHub](https://github.com/macports/macports-guide). Contributors are encouraged to submit pull requests for corrections or updates.

### Workflow Overview

The project uses a triangular workflow for contributions:
1. Fork the main repository
2. Clone the repository locally
3. Create a feature branch
4. Make changes
5. Push changes to your GitHub fork
6. Submit a pull request

### Setting Up the Workflow

#### One-Time Setup Steps:
1. Log in to GitHub
2. Fork the macports-guide repository
3. Clone the main repository to your local machine
4. Install required ports

### Proposing a Change

Detailed steps for contributing:

1. Switch to the `master` branch
2. Pull latest changes
3. Create a new branch with a descriptive name
4. Make documentation changes
5. Validate XML
6. Convert to HTML and proofread
7. Commit changes
8. Push branch to your GitHub fork
9. Create a pull request
10. Monitor and respond to feedback

**Key Guidelines:**
- Make one change or related set of changes per pull request
- Write clear, descriptive commit messages
- Be prepared to discuss and refine your contribution

The full process follows GitHub's [_Forking Projects_](https://guides.github.com/activities/forking/) guide.
